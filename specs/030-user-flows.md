# User Flows

## Основные пользовательские сценарии

### 1. Управление командами

Пользователь открывает `/teams`, создаёт, редактирует и удаляет команды, а также задаёт parent-child структуру.

### 2. Управление периодами

Пользователь открывает `/periods`, создаёт период, редактирует его, двигает вверх / вниз в `sort_order`.

### 3. Просмотр OKR команды

Пользователь открывает `/teams/{teamID}/okr?period_id=...` или `/teamOkrs`, видит:

- команду;
- статус периода;
- суммарный прогресс;
- список goals;
- KR с прогрессом и комментариями.

### 4. Работа с целью

Пользователь:

- создаёт цель;
- редактирует поля цели;
- комментирует цель;
- двигает цель по порядку;
- удаляет цель;
- шарит цель на другие команды;
- меняет weight для owner/shared team.

### 5. Работа с KR

Пользователь:

- создаёт KR;
- редактирует KR;
- обновляет progress;
- комментирует KR;
- меняет порядок KR;
- удаляет KR.

## UX-ограничения из текущего продукта

- Действия целей и KR вынесены в меню.
- Название цели открывает modal edit.
- Добавление KR находится под списком KR.
- При `validated` и `closed` редактирование целей и KR недоступно, но порядок и прогресс ещё доступны.

## Спека на новые UX-фичи

Любая новая UX-фича должна явно указывать:

- на какой странице живёт;
- работает ли через SSR refresh или через API + partial update;
- какие empty / error / loading states обязательны;
- как ведёт себя в `validated / closed`.
